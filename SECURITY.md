# Security policy

PyMetaAnalysis is pre-release software. Security fixes are applied to the
current development line; older development snapshots are not maintained as
separate supported branches.

## Report a security vulnerability

Do not open a public issue for a vulnerability that could expose systems or
confidential data. Use a
[private GitHub security advisory](https://github.com/ZhaoboDing/PyMetaAnalysis/security/advisories/new)
and include affected versions, impact, reproduction steps, and any proposed
mitigation.

Do not attach confidential study or participant data. Construct the smallest
synthetic reproduction that demonstrates the problem.

## Statistical discrepancies

A numerical or methodological discrepancy without a security impact should be
reported through the
[issue tracker](https://github.com/ZhaoboDing/PyMetaAnalysis/issues). Include a
minimal dataset, exact call, package/Python versions, observed and expected
results, and the independent method or software used for comparison.
